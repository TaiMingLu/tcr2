# Scientist Constraints

## Model
The scientist must use a Qwen2.5 model variant (1.5B or 7B) from the Qwen2.5-Instruct family. The paper's contribution is test-time intervention on attention heads — not architecture design. Switching to a different model family (LLaMA, Phi, etc.) would not demonstrate genuine methodological improvement for Qwen2.5 reasoning. The scientist may modify the attention head knockout strategy, the error detection mechanism, or the head selection method — but must use a Qwen2.5-Instruct model.

## Data
The scientist must use the provided synthetic reasoning hop generalization tasks (Parity-NL, LLC, MDM, MOAS, CLF, ObjC, NumS) generated by `data/task_generator.py`. These tasks are deterministic and programmatically generated — no external datasets are needed. No additional training data may be used for the head selector (if implemented), as the paper uses a fine-tuned selector that requires labeled erroneous prediction data.

## Method
The scientist must implement a test-time intervention that modifies LLM behavior without fine-tuning the base model on downstream tasks. The intervention must operate on attention heads or internal representations — not on prompting strategies, decoding parameters, or prompt engineering.

## Compliance Checklist
- [ ] Model is a Qwen2.5-Instruct variant (1.5B or 7B)
- [ ] No external datasets beyond the provided synthetic tasks
- [ ] Test-time intervention (no fine-tuning on downstream tasks)
- [ ] Intervention targets attention heads or internal representations
